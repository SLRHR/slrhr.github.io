SMC Meal Guard

세명컴퓨터고등학교 학생별 알레르기 정보와 급식 데이터를 비교해 위험 메뉴를 알려주는 iOS 앱 프로젝트입니다.

Project Stack

IOS SWIFTUI DATABASE MEAL DATA ALLERGY ALERT

01 · Overview

프로젝트 개요

SMC Meal Guard는 학생이 자신의 알레르기 정보를 등록하면, 급식 메뉴와 성분 정보를 비교해 위험 가능성이 있는 메뉴를 알려주는 식단 관리 앱입니다. 학교 급식 정보를 더 빠르게 확인하고, 알레르기 학생이 안전하게 식단을 판단할 수 있도록 돕는 것을 목표로 합니다.

급식표에는 알레르기 번호가 포함되어 있어도 학생이 매번 직접 확인해야 하고, 본인에게 위험한 메뉴만 따로 구분해 보기 어렵습니다. 이 프로젝트는 개인화된 알레르기 등록과 급식 데이터 분석을 연결해 위험 메뉴를 직관적으로 표시하는 데 초점을 맞췄습니다.

03 · Architecture

구현 구조

Login View

사용자 식별과 개인 알레르기 데이터를 불러오는 진입 화면입니다.

Allergy Profile

학생이 본인의 알레르기 항목을 등록하고 수정합니다.

Meal Data Layer

급식 메뉴와 알레르기 성분 정보를 저장하거나 불러옵니다.

Risk Matching Logic

개인 알레르기 목록과 메뉴 성분을 비교해 위험도를 판단합니다.

Result View

위험 메뉴, 주의 메뉴, 안전 메뉴를 분리해 사용자에게 보여줍니다.

04 · Build Log

어떤 방식으로 만들었는가

iOS 앱 구조는 SwiftUI 화면 단위로 나누는 방식이 적합합니다. 로그인 화면, 메인 화면, 검색 화면, 결과 화면을 분리하고, 데이터베이스에는 학생 정보, 알레르기 정보, 급식 메뉴, 메뉴별 알레르기 성분을 저장하는 구조로 설계할 수 있습니다. 핵심 기능은 급식 메뉴의 알레르기 성분 배열과 학생 알레르기 배열의 교집합을 계산해 위험 메뉴를 판별하는 로직입니다.

급식 메뉴명과 알레르기 성분 표기가 일정하지 않으면 매칭 정확도가 떨어질 수 있습니다. 그래서 메뉴 데이터와 알레르기 데이터를 분리해 저장하고, 화면에서는 최종 결과만 단순하게 보여주는 구조가 필요합니다. 또한 민감한 개인 알레르기 정보가 포함되므로 불필요한 데이터 저장을 줄이고 접근 범위를 명확히 해야 합니다.

05 · Result

결과 및 배운 점

이 프로젝트는 단순 CRUD 앱이 아니라 개인 데이터와 공공성 있는 급식 데이터를 결합해 실제 문제를 해결하는 서비스형 앱입니다. 사용자는 자신의 알레르기 정보만 등록하면 매일 급식에서 주의해야 할 메뉴를 빠르게 확인할 수 있습니다.
